# Billing & KSeF

## Data Model

```
Service             — catalog of billable services (consultation, vaccination, etc.)
Invoice             — one invoice per client per visit
  └─ InvoiceLine    — line items (description, quantity, unit, unit_price_net, vat_rate)
Payment             — payments recorded against an invoice
```

### Invoice statuses

| Status | Meaning |
|---|---|
| `draft` | Created but not sent to client |
| `sent` | Sent to client, awaiting payment |
| `paid` | Fully paid |
| `overdue` | Past due date, not paid |
| `cancelled` | Cancelled |

### KSeF statuses (on Invoice)

| `ksef_status` | Meaning |
|---|---|
| `null` | Not submitted to KSeF yet |
| `pending` | Submission in progress |
| `accepted` | Accepted by KSeF (has `ksef_number`) |
| `error` | Submission failed — can retry |
| `rejected` | Rejected by KSeF (invalid data) |

## API Endpoints

All under `/api/billing/`.

| Method | URL | Description |
|---|---|---|
| GET | `/services/` | List service catalog |
| POST | `/services/` | Create a service |
| GET | `/invoices/` | List invoices (filter: `?status=draft&client=<id>`) |
| POST | `/invoices/` | Create invoice with line items |
| PUT | `/invoices/<id>/` | Update draft invoice |
| POST | `/invoices/<id>/send/` | Mark as sent |
| GET | `/invoices/<id>/payments/` | List payments |
| POST | `/invoices/<id>/payments/` | Record a payment |
| POST | `/invoices/<id>/submit-ksef/` | Build FA3 XML and submit to KSeF |
| GET | `/invoices/<id>/ksef-xml/` | Preview the FA3 XML (debug) |

### Create invoice payload

```json
{
  "client": 1,
  "patient": 2,           // optional
  "status": "draft",
  "due_date": "2026-04-01",
  "currency": "PLN",
  "invoice_number": "FV/2026/03/0001",  // optional, auto-generated if blank
  "lines": [
    {
      "description": "Konsultacja weterynaryjna",
      "quantity": 1,
      "unit": "usł",
      "unit_price": "150.00",           // net price (before VAT)
      "vat_rate": "8",                  // 23 / 8 / 5 / 0 / zw / oo / np
      "service": 3                      // optional: link to Service catalog
    }
  ]
}
```

### VAT rates

| Value | Meaning |
|---|---|
| `23` | 23% — standard rate |
| `8` | 8% — reduced (veterinary services) |
| `5` | 5% — reduced (some food, books) |
| `0` | 0% — zero rate |
| `zw` | Exempt |
| `oo` | Reverse charge |
| `np` | Out of VAT scope |

Veterinary services in Poland typically use **8%**.

## KSeF Integration

### What is KSeF

KSeF (Krajowy System e-Faktur) is Poland's mandatory national e-invoicing system. All B2B VAT invoices must be issued through it.

**Mandatory deadlines:**
- 1 June 2026 — large/medium businesses
- 1 January 2027 — micro-businesses

### How it works in Veto

1. Create an invoice via the Billing tab (or API)
2. Click **"Wyślij do KSeF"** on any invoice row
3. The backend builds a KSeF FA3 XML from the invoice data
4. The XML is submitted to the KSeF REST API
5. On success, the invoice gets a `ksef_number` (KSeF's receipt) and `ksef_status = accepted`

### Configuration

#### Step 1 — Set clinic NIP

In Django Admin: **Tenancy → Clinics → (your clinic) → NIP field** (10 digits, no dashes).

Or via shell:
```python
from apps.tenancy.models import Clinic
c = Clinic.objects.first()
c.nip = "1234567890"
c.save()
```

#### Step 2 — Set KSeF environment

In `config/settings_production.py` (or via environment variable):
```python
KSEF_ENVIRONMENT = "test"   # "test" | "demo" | "prod"
```

- `test` — uses an auto-generated self-signed certificate; no real credentials needed; safe to use for development
- `demo` — KSeF demo environment; requires real certificate
- `prod` — live KSeF; requires real MCU certificate

#### Step 3 (production only) — Set certificate paths

```python
KSEF_CERT_PATH = "/run/secrets/ksef_cert.pem"
KSEF_CERT_KEY_PATH = "/run/secrets/ksef_key.pem"
```

Certificates are issued by the Ministry of Finance (Ministerstwo Finansów) via trusted CAs (MCU).

#### Setting client NIP for B2B invoices

In Django Admin: **Clients → (client) → NIP field**.

If no NIP is set, the invoice is treated as B2C and sent with `BrakIDNab=2` (natural person).

## FA3 XML Structure

Generated by `backend/apps/billing/ksef_xml.py`. Key mappings:

| KSeF field | Source |
|---|---|
| `Podmiot1/NIP` | `invoice.clinic.nip` |
| `Podmiot2/NIP` or `BrakIDNab` | `invoice.client.nip` or B2C |
| `P_1` (issue date) | `invoice.created_at.date()` |
| `P_2` (invoice number) | `invoice.invoice_number` or auto-generated `FV/YYYY/MM/NNNN` |
| `P_7` (line description) | `line.description` |
| `P_8A` (unit) | `line.unit` |
| `P_8B` (quantity) | `line.quantity` |
| `P_9A` (unit net price) | `line.unit_price` |
| `P_11` (line net total) | `line.line_total` |
| `P_12` (VAT rate) | `line.vat_rate` |
| `P_13_*` / `P_14_*` | net and VAT totals by rate |
| `P_15` (gross total) | sum of all line totals + VAT |

## Troubleshooting

**"Clinic NIP is not configured"** — Set the NIP in Django Admin for the clinic.

**"ksef2 is not installed"** — Run `pip install ksef2` (requires Python 3.12+).

**"KSEF_CERT_PATH must be set"** — Either use `KSEF_ENVIRONMENT=test` or provide certificate paths.

**Preview the XML before sending:**
```bash
curl -H "Authorization: Bearer TOKEN" \
  http://localhost:8000/api/billing/invoices/1/ksef-xml/
```
